# Competitor Analysis

## Market Landscape

| Vendor | Model | Strengths | Weaknesses | DA Differentiation |
|--------|-------|-----------|------------|-------------------|
| **Zuub** | SaaS/API verification | Deep benefits, PMS sync | Weak ERA loop | End-to-end workflow |
| **dentalrobot** | Automation + API | 12+ PMS write-back | Less CH-native | Clearinghouse + ack monitoring |
| **Zentist** | DSO cloud RCM | ERA AI, denial triage | Limited PMS posting | DentalBridge write-back breadth |
| **eAssist** | Outsourced billing | Scale via humans | Low software margin | Pure software automation |
| **Dental Intelligence** | Analytics + verification | Broad ops platform | Verification depth | Money-path focus |
| **Vyne Trellis** | CH + RCM | Attachments bundled | Ecosystem lock-in | PMS-agnostic via DentalBridge |
| **DentalXChange** | Clearinghouse | Distribution, trust | Less AI-native | Workflow layer on top |

## Positioning Gaps

1. **Loop closure** — Few competitors own eligibility → post → denial → resubmit
2. **ERA + bank reconciliation** — Underserved vs verification
3. **DSO analytics** — Cross-location denial heatmaps
4. **API product** — DA API as partner surface

## Business Models

- **Per-provider/month** — Zuub, SaaS platforms
- **Per-claim transaction** — Clearinghouses
- **Managed service** — eAssist, DCS (human labor)
- **Hybrid** — Software + exception staffing (DA target)

## MVP Competitive Wedge

ERA posting + eligibility orchestration with mock-first adapters, demonstrating:
- AI extraction + deterministic posting hybrid
- Human-in-the-loop exception queues
- Multi-tenant DSO-ready architecture
